3 CCG categories

3.1 Objects

  • [class] CAT STRUCTURE-OBJECT

    Represent an ABC category. An ABC category is a CCG category with features.

    • NAME represents either the label of an atomic category or the direction of a functor category. Default to ⊥.

    • ARGS is a list of the arguments of this category. Typically they are the antecedent and the consequence of a functor category. Default to empty.

    • FEATS is a fset mapping (key-value pairs) of features. Default to empty.

  • [function] MAKE-CAT &KEY ((:NAME #:NAME) "⊥") ((:ARGS #:ARGS) 'NIL) ((:FEATS #:FEATS) (FSET:EMPTY-MAP))

  • [structure-accessor] GET-NAME

  • [structure-accessor] GET-ARGS

  • [structure-accessor] GET-FEATS

  • [method] MAKE-LOAD-FORM (ITEM CAT)

    From ITEM, create a lisp expression which when evaluated will make the same instance.

3.2 Pattern matchers and destructors

Pattern matchers to use along with [trivia].

  • [symbol-macro] CAT-STR

    Match a CAT with an string representation of another ABC category.

    Usage: (cat-str str unified)

    • STR specifies the string representation to match.

    • UNIFIED stores the result of the unification of the two, from which one can retrieve it.

  • [symbol-macro] CAT-ADJUNCT

    Match an adjunct category. DIR matches its functor type. RADICAL matches its categorial radical.

  • [function] UNCURRY-CAT ITEM &KEY (NAME-P T)

    List the antecedents of a (possibly multiple) functor category ITEM.

    The primary returning value is, if any, the outmost functor name. The secondary returning value is a list of functor antecedents. The tertiary value is the consequence.

    NAME-P specifies the functor ('/', '\', or '|') the function should exclusively match with. For example, if NAME-P is set as '/', from <<C\<A/B>>/D>/E the function yields (E D) and <C\<A/B>> (the search stops at the '\' functor).

    If NAME-P is NIL, any functor categories will make a match. If NAME-P is T, the first found functor is used.

  • [symbol-macro] CAT-UNCURRIED-IGNORE-FUNCTORS

    Match a CAT with all of its functors destructed regardless of their types. ARGS, as a list, stores the functor categorie(s). CONSEQ stores the consequent category.

    Example: <A\B\C\D>/E makes a match with (cat-uncurried-ignore-functors args conseq), where

    • ARGS is (list E A B C), and

    • CONSEQ is D.

  • [symbol-macro] CAT-UNCURRIED

    Match a CAT with its functors destructed. FUNCTOR stores the functor type. ARGS, as a list, stores the functor categorie(s). CONSEQ stores the consequent category.

    Example: A/B/C/D/E makes a match with (cat-uncurried functor args conseq), where

    • FUNCTOR is '/',

    • ARGS is (list E D C B), and

    • CONSEQ is A.

3.5.2 Reduction

  • [function] REDUCE-CAT CAT-LEFT CAT-RIGHT

    Attempt an →-elimnation (/-elimination, -elimination, |-elimination) or a functional application on CAT-LEFT and CAT-RIGHT.

    • The first returning value is the resulting category. NIL in case of failure.

    • The second returning value is of type REDUCE-RESULT and contains detailed information about the rule that is applied. NIL in case of failure.

The following is the definitions of REDUCE-RESULT and related objects.

  • [class] REDUCE-RESULT STRUCTURE-OBJECT

    Stores a detail result of REDUCE-CAT.

  • [function] MAKE-REDUCE-RESULT &KEY ((:REDUCTION #:REDUCTION) "") ((:LEVEL #:LEVEL) 0)

  • [structure-accessor] GET-REDUCTION

  • [structure-accessor] GET-LEVEL

  • [function] SERIALIZE-REDUCE-RESULT ITEM

5 Trees

[in package AMOOVE/PSD]

A tree in [amoove] is just a lisp list in which the first element is the label of the tree, which is followed by subtrees as lisp lists again.

← ↑ → ↺ λ

5.1 Parsing

  • [function] GET-PARSER STRM &KEY (BUF-SIZE 100)

    Generate an ABC tree parser on the fly. BUF-SIZE is the size of the buffer that holds nodes.

5.2 Serialization

  • [function] PPRINT-TREE TREE &KEY (CONVERTER (LAMBDA (I) I)) (OUTPUT-STREAM *STANDARD-OUTPUT*) (ID NIL) (INDENT 0) (IS-MULTI-LINES T) (IS-SECOND-OR-LATTER-CHILDREN NIL) (ALIGN-MULTIBYTE NIL)

    Pretty print TREE.

    • CONVERTER speficies how to pretty print nodes.

    • OUTPUT-STREAM speficies the output stream.

    • ID specifies an ID of the given tree. If it is not NIL, the tree is wrapped with that ID.

    • INDENT specifies the indent of this (sub)tree.

  • [function] EXPORT-AS-REACT-D3-TREE TREE &KEY (ID NIL) (CONVERTER (LAMBDA (I) I)) (OUTPUT-STREAM *OUTPUT-STREAM*)

5.3 Manipulation functions

  • [function] SPLIT-COMMENTS TREE &KEY (NODE-PRED (LAMBDA (I) (STRING-EQUAL I "COMMENT")))

    Tease out 'COMMENT' nodes from TREE.

    NODE-PRED specifies comment node labels.

    The function returns a pair of the tree without comments and the comments teased apart.

5.4 Errors

  • [condition] PENNTREE-PARSE-ERROR AMOOVE:BASE-ERROR

    An error raised when an error occurs during parsing a tree.

The following errors are subclasses of PENNTREE-PARSE-ERROR.

  • [condition] UNEXPECTED-TOKEN PENNTREE-PARSE-ERROR

    An error raised when the parser meet an unexpected token.

  • [condition] UNCLOSED-PARENTHESIS PENNTREE-PARSE-ERROR

    An error raised when the parser finds EOF with an parenthesis being unclosed.
